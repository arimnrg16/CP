Deskripsi

Untuk persiapan agar dapat diterima kuliah di Universitas CFX, Pak Chanek sekarang sedang mengajari Chaneka yang baru berumur 10 tahun tentang sifat-sifat suatu string. Pak Chanek memberikan Chaneka sebuah string S yang mempunyai panjang N dan hanya berisi huruf alfabet kecil ('a' sampai 'z') saja, karakter terkiri dari S adalah karakter ke-1 dan karakter terkanan dari S adalah karakter ke-N. Pak Chanek mempunyai 2 jenis perintah, yaitu:

  1. Mengubah karakter ke-X pada string S menjadi C;
  2. Menghitung ada berapa banyak substring berbeda dari S[L..R] yang mana tidak ada karakter yang sama muncul lebih dari 1 kali, dengan S[L..R] adalah substring S yang dimulai dari karakter ke-L sampai karakter ke-R.

Dua buah substring dari S[L..R] (S[L1..R1] dan S2[L2..R2] dengan L1 <= R1; L2 <= R2; L1, L2 >= L; dan R1, R2 <= R) dianggap berbeda jika L1 ≠ L2 atau R1 ≠ R2.

Pak Chanek memberi perintah-perintah tersebut kepada Chaneka sebanyak Q perintah dan Chaneka diminta untuk melakukan perintah tersebut dengan cepat. Karena Chaneka kelelahan, ia meminta Anda untuk membuatkan program untuk menyelesaikan perintah tersebut. Bantulah Chaneka!
 

Format Masukan

Baris pertama berisi 2 buah bilangan N dan Q, panjang dari string S dan banyaknya perintah yang diberikan.

Baris kedua berisi sebuah string S.

Q baris berikutnya berisi salah satu dari berikut:

  • 1 X C, yang berarti ubah karakter ke-X menjadi C.
  • 2 L R, yang berarti hitung ada berapa banyak substring berbeda dari S[L..R] yang mana tidak ada karakter yang sama muncul lebih dari 1 kali, dengan S[L..R] adalah substring S yang dimulai dari karakter ke-L sampai karakter ke-R.

Format Keluaran

Untuk setiap perintah tipe 2, keluarkan sebuah baris yang berisi hasil perhitungan dari perintah tersebut. Dikarenakan hasilnya bisa cukup besar, keluarkan hasil modulo 109 + 7.

Contoh Masukan

8 3
compfest
2 4 8
1 3 o
2 1 5

Contoh Keluaran

15
9

Penjelasan Contoh

Untuk perintah pertama, substring-substring dari S[4..8] yang tidak mengandung lebih dari 1 karakter yang sama adalah:

  1. S[4..4] = p
  2. S[4..5] = pf
  3. S[4..6] = pfe
  4. S[4..7] = pfes
  5. S[4..8] = pfest
  6. S[5..5] = f
  7. S[5..6] = fe
  8. S[5..7] = fes
  9. S[5..8] = fest
  10. S[6..6] = e
  11. S[6..7] = es
  12. S[6..8] = est
  13. S[7..7] = s
  14. S[7..8] = st
  15. S[8..8] = t

Maka, jawaban dari perintah pertama adalah 15.

Untuk perintah kedua, string S menjadi 'coopfest'.

Untuk perintah ketiga, substring-substring dari S[1..5] yang tidak mengandung lebih dari 1 karakter yang sama adalah:

  1. S[1..1] = c
  2. S[1..2] = co
  3. S[2..2] = o
  4. S[3..3] = o
  5. S[3..4] = op
  6. S[3..5] = opf
  7. S[4..4] = p
  8. S[4..5] = pf
  9. S[5..5] = f

Maka, jawaban dari perintah ketiga adalah 9.
 

Subsoal

Untuk semua subsoal, berlaku:

  • 1 ≤ N, Q ≤ 100.000
  • String S dijamin hanya berisi karakter dari 'a' sampai 'z'
  • Untuk setiap perintah tipe 1, 1 ≤ X ≤ N dan C dijamin berupa sebuah karakter dari 'a' sampai 'z'
  • Untuk setiap perintah tipe 2, 1 ≤ L ≤ R ≤ N

Subsoal 1 (7 poin)

  • Dijamin tidak ada perintah tipe 1 pada masukan
  • Huruf-huruf pada string S terurut secara alfabetik

Subsoal 2 (23 poin)

  • 1 ≤ N, Q ≤ 1.000

Subsoal 3 (25 poin)

  • Dijamin tidak ada perintah tipe 1 pada masukan

Subsoal 4 (45 poin)

  • Tidak ada batasan khusus

 

Catatan

Suatu string S dikatakan terurut secara alfabetik jika dan hanya jika Si ≤ Si+1 untuk 1 ≤ i < N, yang artinya karakter ke-i pada S muncul lebih dulu dari karakter ke-(i+1) pada urutan alfabet.
